High Performance Canny Edge Detection

CMU 18645 - How to Write Fast Code II

Canny Edge Detection is one of the most well-known algorithms developed by John F. Canny to detect a wide range of edges in images. This algorithm can find the edge precisely. However, the algorithm is inefficient when processing an image on CPUs, even though each pixel is processed independently. To address this issue, paralleling the image processing on each pixel can enhance the algorithm’s performance.

In this project, we will try to optimize the performance of Canny Edge Detection by using SIMD instructions and OpenMP to process pixels simultaneously.

For the details of the design or some snippets of the code, please email me. I can share some useful thoughts.

3D Human Pose Estimation System

BJTU Undergraduate Project
Enable to lift 3D Human skeleton pose from RGB 2D image.
  • Unsupervised learning: It's not even necessary for you to feed this model 2D labeled data, just the 2D joints will get all net worked.
  • 3D data FREE: You don't even need the 3D data to train your model, by firstly generate the 3D pose and secondly generate the 3D pose by transformation and inverse transformation, then get MPJPE loss from these two 3D pose will eliminate the scarecity of 3D data.
  • Flexibility: Someone might wonder that what if I got some 3D pose data? Congratulations! You can absolutely firstly compute the 3D regression loss to get the model "warm start", that will help a lot.
  • Avoid OverfittingBy using discriminator to fit the normal distribution of normal human pose and geometrical constraints of human limb symmetry, some traits learned by the model can be robust to multiple types of data.
